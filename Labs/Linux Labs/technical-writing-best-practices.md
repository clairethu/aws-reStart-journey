
## ✍️ Documentation Best Practices: Markdown Code Blocks
*Ensuring technical logs remain readable and professional.*

- **Rule:** Always use opening and closing "Triple Backticks" (\` \` \`) to wrap code snippets.
- **Syntax Highlighting:** Adding `bash` after the first set of backticks tells GitHub to color-code the command logic for better visibility.
- **The "Switch" Principle:** The second set of backticks acts as a closing tag to return the document to standard prose formatting.

> **PM Insight:** Clean documentation is just as important as clean code. In a project hand-off, well-formatted logs ensure that stakeholders and future engineers can quickly parse and understand the work performed without distraction.
